EOS Event UNDP di Royal Ambarrukmo Hotel

10 Jan 2026
By Rosyid Majid

Gambaran Proyek

Event UNDP membutuhkan jaringan dengan performa tinggi dan keandalan maksimal untuk mendukung ratusan peserta dan staf. Area venue dibagi menjadi tiga zona, masing-masing dilayani oleh Access Point Ruijie (AP): Zona Utara, Tengah, dan Selatan.

Monitoring Jaringan:

  • Pemantauan real-time menggunakan Winbox pada router MikroTik untuk bandwidth dan latensi.
  • AP dikelola secara remote melalui Ruijie Cloud Controller.
  • Memungkinkan deteksi dan penanganan masalah jaringan secara cepat, sehingga konektivitas tetap stabil selama acara berlangsung.

Tanggung Jawab

  • Monitoring Jaringan: Memantau performa setiap AP dan keseluruhan jaringan secara berkelanjutan.
  • Respons Insiden: Mengambil tindakan cepat ketika terjadi gangguan jaringan.

Studi Kasus: Permasalahan & Tindakan

1. AP Zona Utara Offline

Kategori Detail
Masalah Access Point Ruijie di Zona Utara mengalami offline sehingga koneksi pengguna terputus.
Tindakan Restart AP secara remote melalui Ruijie Cloud Controller dan dilakukan monitoring hingga AP kembali online.
Hasil AP berhasil dipulihkan tanpa memengaruhi Zona Tengah dan Selatan, dengan downtime yang minimal.

2. Bandwidth Penuh di Seluruh Zona

Kategori Detail
Masalah Penggunaan bandwidth mencapai batas maksimal di semua zona, menyebabkan koneksi lambat dan packet loss.
Tindakan Melakukan penyesuaian Queue Tree pada router MikroTik untuk menyeimbangkan trafik antar zona.
Hasil Jaringan kembali stabil, latensi menurun, dan seluruh zona tetap terhubung optimal saat traffic tinggi.

Tools & Teknologi

  • Router MikroTik (Queue Tree + monitoring via Winbox)
  • Access Point Ruijie (Zona Utara, Tengah, Selatan, dikelola melalui Ruijie Cloud Controller)
  • Script monitoring untuk pengecekan ping, latensi, dan bandwidth

Analisis Jaringan & Insight

  • Pola Trafik Puncak: Zona Tengah menunjukkan penggunaan bandwidth tertinggi karena aktivitas presentasi dan streaming peserta.
  • Keandalan AP: Ruijie Cloud Controller memungkinkan deteksi AP offline dengan cepat, menurunkan rata-rata downtime menjadi di bawah 5 menit.
  • Manajemen Bandwidth: Konfigurasi Queue Tree terbukti efektif mengurangi kemacetan dan meningkatkan stabilitas jaringan secara keseluruhan.
  • Observasi Latensi: Monitoring ping berkelanjutan mendeteksi lonjakan latensi kecil saat jam sibuk, yang langsung diatasi dengan penyeimbangan trafik.

Hasil & Pembelajaran

  • Performa jaringan stabil selama event dengan downtime yang sangat minim.
  • Respons cepat terhadap gangguan AP maupun masalah bandwidth.
  • Menunjukkan kemampuan monitoring jaringan real-time dan troubleshooting pada event berskala besar dan multi-zona.
  • Pelajaran penting:
    • Monitoring real-time dan manajemen AP berbasis cloud sangat krusial untuk jaringan event besar.
    • Manajemen trafik yang proaktif mencegah congestion dan meningkatkan pengalaman pengguna.
    • Pemahaman pola trafik tiap zona membantu perencanaan kapasitas untuk event berikutnya.
